Introduction to JavaScript

JavaScript is a high-level, interpreted programming language that is one of the core technologies of the World Wide Web. It enables interactive web pages and is an essential part of web applications.

Initially created to make web pages alive, JavaScript can execute in the browser, on the server, or on any device that has a JavaScript engine.

console.log("Hello, World!");

JavaScript follows the ECMAScript specification and has evolved significantly since its creation in 1995.

  • Client-side scripting for web browsers
  • Server-side development with Node.js
Variables and Data Types

Variables in JavaScript are containers for storing data values. JavaScript uses keywords like var, let, and const for variable declaration.

let message = "Hello";
const PI = 3.14159;
var count = 10;

JavaScript has dynamic typing, meaning a variable can hold any type of data without explicit type declaration.

Primitive data types include:

  • String: Textual data
  • Number: Integer or floating-point numbers
  • Boolean: true/false values
  • Undefined: Variable declared but not assigned
  • Null: Intentional absence of value
let name = "Alice"; // String
let age = 30; // Number
let isStudent = true; // Boolean
Functions and Scope

Functions are reusable blocks of code that perform specific tasks. They can take parameters and return values.

JavaScript has function scope and block scope (with let and const). Variables declared with var have function scope.

function greet(name) {
return "Hello, " + name + "!";
}

// Arrow function
const multiply = (a, b) => a * b;

Closures are an important concept where a function retains access to variables from its lexical scope.

Hoisting is JavaScript's behavior of moving declarations to the top of their containing scope during compilation.

// Function declaration (hoisted)
sayHello();

function sayHello() {
console.log("Hello!");
}
Control Structures

Control structures determine the flow of program execution. JavaScript supports various conditional statements and loops.

Conditional statements include if, else if, else, and switch.

let score = 85;

if (score >= 90) {
console.log("A");
} else if (score >= 80) {
console.log("B");
} else {
console.log("C");
}

Loop structures include for, while, do...while, and for...of/for...in loops.

// For loop
for (let i = 0; i < 5; i++) {
console.log(i);
}

// While loop
let i = 0;
while (i < 5) {
console.log(i);
i++;
}
Objects and Arrays

Objects are collections of key-value pairs used to store complex data structures. Arrays are list-like objects used to store multiple values.

// Object
const person = {
name: "John",
age: 30,
profession: "Developer"
};

// Array
const colors = ["red", "green", "blue"];

Objects can have properties and methods. Properties are values associated with an object, and methods are functions that belong to an object.

Arrays have many built-in methods like push(), pop(), map(), filter(), and reduce().

  • Objects use dot notation or bracket notation
  • Arrays are zero-indexed
  • Both can be nested and combined
// Accessing object properties
console.log(person.name); // "John"
console.log(person["age"]); // 30

// Array methods
colors.push("yellow");
console.log(colors.length); // 4
DOM Manipulation

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page so programs can change the document structure, style, and content.

JavaScript can access and manipulate the DOM to create dynamic web pages.

// Select element
const element = document.getElementById("myElement");

// Change content
element.textContent = "New content";

// Add event listener
element.addEventListener("click", function() {
alert("Element clicked!");
});

Common DOM manipulation methods include:

  • getElementById(), querySelector()
  • createElement(), appendChild()
  • setAttribute(), removeAttribute()
  • classList.add(), classList.remove()

Event handling allows JavaScript to respond to user interactions like clicks, keypresses, and form submissions.

Error Handling

Error handling in JavaScript is done using try...catch statements. This allows you to handle runtime errors gracefully without stopping script execution.

try {
// Code that might throw an error
let result = riskyOperation();
console.log(result);
} catch (error) {
// Handle the error
console.log("An error occurred:", error.message);
} finally {
// Code that runs regardless of error
console.log("Operation attempted");
}

JavaScript has several built-in error types including SyntaxError, ReferenceError, TypeError, and RangeError.

You can also throw custom errors using the throw statement with any value, but it's common to use Error objects.

function validateInput(input) {
if (!input) {
throw new Error("Input cannot be empty");
}
return input;
}

Proper error handling improves user experience and makes debugging easier.
